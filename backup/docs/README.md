# 📚 知识库 (Knowledge Wiki)

> 个人开发知识库，记录所有迭代更新、功能开发、脚本创建和Bug修复。

## 🗂️ 文档结构

```
docs/
├── README.md           # 本文件 - 知识库入口
├── CHANGELOG.md        # 变更总览 (时间线)
├── changelog/          # 详细变更记录 (按日期/版本)
│   ├── 2026-02-04.md   # 每日/每次迭代的详细记录
│   └── ...
├── wiki/               # 知识Wiki (按主题分类)
│   ├── scripts.md      # 脚本使用指南
│   ├── skills.md       # AI Skills 文档
│   ├── troubleshooting.md  # 问题排查指南
│   └── ...
└── templates/          # 文档模板
    ├── changelog-entry.md  # 变更记录模板
    └── wiki-article.md     # Wiki文章模板
```

## 🏷️ 变更类型标签

| 标签 | 说明 | 使用场景 |
|------|------|----------|
| `✨ FEATURE` | 新功能 | 新增功能或特性 |
| `🔧 UPDATE` | 迭代更新 | 现有功能的改进或升级 |
| `🐛 BUGFIX` | Bug修复 | 修复已知问题 |
| `📜 SCRIPT` | 脚本创建 | 新工具脚本 |
| `📖 DOCS` | 文档更新 | 文档新增或修改 |
| `🔨 REFACTOR` | 重构 | 代码重构，不改变功能 |
| `⚡ PERF` | 性能优化 | 性能相关改进 |

## 🔍 如何使用

### 记录新变更
1. 在 `changelog/` 目录下创建或编辑对应日期的文件
2. 使用 `templates/changelog-entry.md` 模板
3. 更新 `CHANGELOG.md` 总览

### 添加知识文档
1. 在 `wiki/` 目录下创建新的主题文件
2. 使用 `templates/wiki-article.md` 模板
3. 在本文件的快速导航中添加链接
