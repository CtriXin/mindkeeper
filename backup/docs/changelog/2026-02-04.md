## 🔧 UPDATE: Domain工具Excel映射和预览功能优化

> **摘要**: 修改domain工具的config.js配置文件，适配新的Excel表头格式，并优化--preview功能只输出实际修改的域名

### 📋 基本信息

| 项目 | 内容 |
|------|------|
| **变更类型** | `🔧 UPDATE` |
| **影响范围** | domain-service-lookup工具 |
| **相关文件** | `config.js`, domain工具相关脚本 |
| **耗时** | 约 2 小时 |

### 🎯 背景与目标

**问题/需求描述：**
- Excel表头格式发生变化，需要更新列映射配置
- `--preview` 功能输出了所有域名，而不是只输出本次修改的域名
- 需要将预览结果输出到指定文件 `preview-output.json`

**期望目标：**
- 更新Excel列到JSON字段的映射
- 修改输出模板，确保只处理相关字段
- `--preview` 只显示实际被修改的域名
- 预览结果输出到 `preview-output.json` 文件

### 🔄 实现过程

#### 1. 分析新的Excel表头格式
- 确认新的列名称和对应的JSON字段
- 识别需要保留和需要移除的字段

#### 2. 更新config.js配置
- 修改 `excelToJsonMapping` 映射关系
- 调整输出模板 (template) 结构
- 更新字段过滤逻辑

#### 3. 优化--preview功能
- 修改预览逻辑，只收集实际发生变化的域名
- 添加差异对比功能
- 将结果输出到 `preview-output.json`

### ❗ 遇到的问题

#### 问题 1: 预览输出包含所有域名
- **现象**: 使用 `--preview` 时，输出了Excel中所有域名，而非仅修改的
- **原因**: 原逻辑遍历所有行并输出，没有进行变更检测
- **解决方案**: 添加变更检测逻辑，对比修改前后的值，只记录有差异的项

#### 问题 2: 字段映射不匹配
- **现象**: 某些字段无法正确从Excel读取
- **原因**: Excel列名包含空格或特殊字符，与配置中的key不一致
- **解决方案**: 规范化列名处理，去除多余空格

### ✅ 最终结果

- [x] Excel列映射更新完成
- [x] 输出模板调整完成
- [x] --preview 只输出修改的域名
- [x] 预览结果正确输出到 preview-output.json

**验证方式：**
```bash
# 使用预览模式测试
domain --preview --input test.xlsx
# 检查 preview-output.json 内容
cat preview-output.json
```

### 📝 经验总结

- Excel列名处理需要注意空格和特殊字符
- 预览功能应该明确只显示"变化"，而不是"全部"
- 配置文件的结构设计会影响后续维护的便利性
