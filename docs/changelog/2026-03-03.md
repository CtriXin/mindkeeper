# 🏷️ [📜 SCRIPT] Git Worktree 交互式管理工具 (`wt`) 开发

> **摘要**: 开发了一个支持 TUI 箭头选择、按时间倒序匹配、以及自动处理分支冲突的 Git worktree 辅助脚本 `bin/wt`。

### 📋 基本信息
| 项目 | 内容 |
|------|------|
| **变更类型** | `📜 SCRIPT` |
| **影响范围** | `bin/`, Git 工作流 |
| **相关文件** | `bin/wt`, `GIT_SETUP.md`, `.gitignore` |

### 🎯 背景与目标
由于原生 `git worktree` 命令繁琐且难以记忆分支上下文，用户需要一个：
1. 能按时间倒序推荐分支的工具。
2. 支持模糊搜索和 TUI 交互（箭头选择）。
3. 能自动记录 worktree 中文描述并合并显示。
4. 能自动处理「分支已在其他工作区检出」冲突的自动化方案。

### 🔄 实现过程
1. **规划阶段**：确定了 `bin/wt` 子命令结构（create, list, remove, branch）。
2. **核心开发**：
   - 使用 `git for-each-ref` 实现了分支的按时间倒序展示。
   - 开发了 `interactive_select` 纯 Bash TUI 函数，通过截获 ANSI 转义序列实现箭头按键导航。
   - 实现了 `.worktree-meta.json` 本地元数据持久化，用于存储中文描述。
3. **兼容性优化**：适配 macOS 自带的 Bash 3.2 版本，修复了 `read -t` 浮点数超时和 `trap` 信号兼容性问题。
4. **冲突自动化**：引入了自动检测机制，冲突时自动附加 `_wt_N` 后缀（如 `main_wt_1`）直接检出。

### ❗ 遇到的问题
- **问题**: macOS 上 `read -rsn2` 无法正确捕捉方向键。
- **解决方案**: 改为循环读取 `read -rsn1` 并配合 `key2` `key3` 变量解析。

### ✅ 最终结果
- `bin/wt` 可直接运行开启交互模式。
- 所有功能已在 `feature/worktree-management` 分支上完成测试并落地。

### 📝 经验总结
- 在旧版 Bash (3.2) 下做 TUI 交互时，必须严格遵守 `read` 的 POSIX 限制，不能使用超过 1 秒的分数超时探测。
- `git show-ref --verify` 是检测分支是否存在的最高效方式。
