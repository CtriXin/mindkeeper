# 你的 AI 编程助手，每次对话都失忆？

<!-- 封面图：docs/cards/01-hero.png -->

---

用 Claude Code / Cursor 写代码的朋友，有没有遇到这种情况：

❌ 上午写了个复杂功能，下午继续的时候 AI 完全不记得了
❌ 上周在 A 项目踩过的坑，这周在 B 项目又踩了一遍
❌ 让 AI 帮忙接着昨天的进度，结果它从零开始
❌ 同一个问题，重复解释了三遍，AI 还是不记得

这不是 AI 不聪明，是它真的**没有记忆**。

每次新 Session = 一个全新的大脑，之前的一切都不存在。

<!-- 插图1：docs/cards/02-workflow.png -->

每天早上打开 Claude Code 的第一件事是什么？

「继续昨天的工作。」

然后你发现——AI 不记得昨天做了什么。

你开始手动复述：
- 「昨天改了 auth.ts 和 middleware.ts」
- 「决定用 JWT 替代 session cookie」
- 「发现 iOS Safari 有兼容问题」
- 「还没写测试」

每天花 10-15 分钟**给 AI 补课**，一周就是一个多小时。

---

## 我做了个工具解决这个问题

**MindKeeper** — 给 AI Agent 装一个跨 Session 的第二大脑。

它是一个 MCP Server，装上之后：

✅ 上次做过类似的？**自动推送已有方案**
✅ 昨天的进度？**30 秒恢复到上次状态**
✅ 跨项目的坑？**一次踩坑，所有项目受益**
✅ 经验会衰减？**90 天不用自动降权，保持知识库精准**

不是那种「存一堆 markdown 然后全塞给 AI」的粗暴方案。
是**结构化的知识管理 + 语义匹配 + 主动推送**。

---

## 一行命令安装

```bash
curl -fsSL https://raw.githubusercontent.com/CtriXin/mindkeeper/main/install.sh | bash
```

或者直接把这句话粘贴给你的 AI Agent，让它自己装自己的大脑：

```
Install MindKeeper MCP server by running: curl -fsSL https://raw.githubusercontent.com/CtriXin/mindkeeper/main/install.sh | bash — it handles everything including MCP config. Restart the client after install.
```
