# AI 的知识库越大越好？不，会越用越差

<!-- 封面图：docs/cards/01-hero.png 或 mindkeeper-overview-cn.png -->

---

很多人做 AI 记忆系统的思路：「全存下来，越多越好。」

几个月后：
- 知识库堆了几百条
- 搜索结果一半是过时的
- Vue2 的经验推荐给了 Vue3 项目
- jQuery 的方案混在 React 里

**信息越多，噪音越大，AI 反而更容易犯错。**

---

## MindKeeper 的解法：自动衰减

<!-- 插图1：recipe-lifecycle.excalidraw 导出图 -->

借鉴人脑遗忘曲线 — 常用的强化，不用的自然淡出。

### Recipe 的四个阶段

**诞生** → brain_learn 提取，confidence 0.9
**活跃** → 被频繁 recall，accessCount 持续增长
**过时** → 90 天没用，标记 stale
**降级** → 180 天 + 0 次使用，confidence 降到 0.3

**降级不是删除**——被重新使用会复活，恢复权重。

## 三域架构

<!-- 插图2：integration-architecture.excalidraw 导出图 -->

不是缝合怪，是从零设计的三域闭环：

**Recipe 知识库** — 结构化经验，语义匹配，跨项目复用
**Thread 蒸馏** — 工作状态快照，30秒恢复，自动 GC
**Board 看板** — 四象限任务管理，信号自动检测

三域怎么关联？
- checkpoint 保存 Thread → 自动关联 Board item
- Board item 完成 → 提示提取 Recipe
- learn 提取 Recipe → 自动关联 Board item

**闭环：工作 → 看板 → 知识 → 下次工作。**

---

## 为什么不用向量数据库？

**零依赖** — 不需要 Chroma / Pinecone / 任何服务
**可审查** — 打开文件就能看到 AI 学了什么
**可编辑** — 用户直接修改 Recipe
**够用** — 个人知识库规模下，规则匹配足够精准

所有数据存在 `~/.sce/`，纯 Markdown + JSON，人可读，Git 友好。

---

## MindKeeper 的第一性价值

> 帮你在 30 秒内进入正确工作状态。
> 不是帮你记住更多，是帮你精准地记住该记住的。
