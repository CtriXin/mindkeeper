# Promotion Checklist

Use this before moving a lesson into stable notes or the main skill.

## Promote to `scmp-ops/SKILL.md` only if all are true

- repeated at least `2-3` times
- useful across projects
- concrete and testable
- forgetting it would likely cause future production mistakes
- short enough to live in a stable SOP

If any item is false, do **not** promote to the main skill yet.

## Promote to `ops-notes.md` or `ad-bug-notes.md` if most are true

- reusable
- validated
- too detailed for main SOP
- likely to help again soon

## Keep only in inbox if any are true

- happened once
- still ambiguous
- only applies to one temporary DOM or one one-off page
- mostly a personal preference
- no solid runtime or command evidence yet

## Quick classification matrix

- **Main skill**
  - short, default, cross-project, high-cost-to-forget
- **Reference notes**
  - reusable, detailed, validated
- **Inbox**
  - fresh, promising, not yet proven
- **Recipe/memory only**
  - one-off, project-local, temporary
